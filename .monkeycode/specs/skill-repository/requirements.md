# 技能仓库需求

## 介绍

技能仓库为 TokenHub 管理后台提供一级入口，用于离线保存、查看、导入和删除符合 `SKILL.md` 约定的技能包，并从公开技能来源搜索可导入的技能。技能仓库将 GitHub、SkillHub 等公开来源统一为可扩展的搜索来源；管理员确认后，系统下载并校验技能包，再写入本地仓库。

## 术语

- **技能仓库**：服务端管理的本地 Skill 文件集合。
- **技能包**：包含根目录 `SKILL.md` 的目录及其关联文件。
- **本地技能**：已保存于技能仓库目录的技能包。
- **公开来源**：无需身份认证即可查询或下载的 Skill 索引或代码托管网站。
- **来源适配器**：负责向一个公开来源查询、解析和下载内容的服务组件。
- **搜索结果**：公开来源返回并转换为统一结构的技能元数据。
- **导入**：管理员确认后，将公开技能包下载、验证并写入本地技能仓库的过程。
- **冲突**：导入目标路径已存在本地技能包的状态。

## 需求

### 需求 1：本地技能仓库

**用户故事：** 作为管理员，我希望集中查看和管理本地 Skill 文件，以便离线使用和维护技能包。

#### 验收标准

1. 系统应当提供一级功能“技能仓库”。
2. 当管理员进入“技能仓库”时，系统应当展示本地技能包列表、搜索入口和导入入口。
3. 当服务启动时，系统应当扫描配置的技能仓库目录，并以包含根目录 `SKILL.md` 的目录识别技能包。
4. 当管理员查看本地技能包时，系统应当展示名称、描述、目录名、来源、导入时间和文件数量。
5. 当技能包的 `SKILL.md` 缺失或无法解析时，系统应当展示该目录的校验状态和诊断信息。
6. 当管理员选择本地技能包时，系统应当展示 `SKILL.md` 内容及安全的文件清单。

### 需求 2：离线导入与文件管理

**用户故事：** 作为管理员，我希望导入和维护本地 Skill 文件，以便在无网络连接时继续使用。

#### 验收标准

1. 当管理员上传技能包归档文件时，系统应当校验归档内存在唯一的技能根目录和 `SKILL.md`。
2. 当管理员从公开搜索结果导入技能包时，系统应当先展示名称、描述、来源地址、提交版本和文件清单，再等待管理员确认。
3. 当管理员确认导入时，系统应当将技能包写入配置的本地技能仓库目录，并记录来源元数据和导入时间。
4. 当导入目标目录已存在本地技能包时，系统应当展示冲突详情并要求管理员选择保留本地版本或替换为导入版本。
5. 当管理员删除本地技能包时，系统应当先展示待删除目录和文件数量，并在管理员二次确认后移除该技能包。
6. 当技能包包含符号链接、绝对路径、路径穿越条目或超出配置限制的文件时，系统应当拒绝导入并说明校验原因。

### 需求 3：公开来源搜索

**用户故事：** 作为管理员，我希望从公开技能网站搜索 Skill，以便发现和导入可复用能力。

#### 验收标准

1. 当管理员输入关键词时，系统应当查询启用的公开来源，并返回统一格式的搜索结果。
2. 系统应当提供 GitHub 和 SkillHub 作为首批公开来源适配器。
3. 当管理员配置公开 Skill 索引地址时，系统应当将该地址作为可启用的公开来源。
4. 当公开来源返回结果时，系统应当展示来源名称、技能名称、描述、作者或组织、更新时间、星标或热度信息、许可证信息和来源地址中的可用字段。
5. 当某个公开来源请求失败时，系统应当保留其他来源的结果，并展示该来源的失败状态。
6. 当公开来源需要身份认证时，系统应当将该来源标记为当前不可用，并继续处理匿名可访问的来源。
7. 当管理员选择搜索结果时，系统应当获取技能包文件清单并验证 `SKILL.md` 后提供导入确认界面。

### 需求 4：来源配置

**用户故事：** 作为管理员，我希望控制搜索来源及其启用状态，以便获得符合团队标准的搜索结果。

#### 验收标准

1. 当管理员打开来源配置时，系统应当展示 GitHub、SkillHub 和自定义公开索引的名称、地址、启用状态和最近同步状态。
2. 当管理员添加自定义公开索引时，系统应当验证地址使用 HTTPS，并验证返回内容符合支持的公开索引格式。
3. 当管理员启用或停用公开来源时，系统应当在后续搜索请求中应用新的来源集合。
4. 当管理员修改来源配置时，系统应当持久化保存配置，并在服务重启后恢复配置。

### 需求 5：可追溯性与权限

**用户故事：** 作为管理员，我希望了解本地技能的来源与操作记录，以便安全维护技能仓库。

#### 验收标准

1. 系统应当要求管理员会话访问技能仓库接口和页面。
2. 当系统导入技能包时，系统应当保存来源类型、来源地址、版本标识、导入时间和内容摘要。
3. 当管理员执行导入、替换或删除操作时，系统应当记录操作时间、操作类型、技能目录和执行结果。
4. 当管理员查看本地技能包时，系统应当展示最近一次导入或替换操作的来源信息。
